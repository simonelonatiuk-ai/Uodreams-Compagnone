1.1 Scripting

Welcome to the script module for OrionUO!

Used script engine: JavaScript

WARNING!

Be careful when using protected include files (*.jsp and *.oajsp) received from unfamiliar people.

Such files can harm your computer and character.

Available classes:

- Orion - the main class through which interaction with the world takes place.

- Player - player class, used for faster access to player properties.

- SelectedTile - the tile class in the world above which the mouse cursor is located.

- TextWindow - Assistant text window class.

- Abyss - class for working with robes/cloaks that have spell charges. For The Abyss server only.

- GameObject - game object class (item/corpse/living creature).

- JournalMessage - journal message class.

- PositionObject - coordinate class (X, Y, Z).

- GumpObject - class of the gump object received from the server.

- MenuObject - the class of the menu object received from the server.

- GumpHookObject - class hooks for gumps.

- ClientMacroObject - class for creating and playing client macros.

- TileObject - map tile object class (land, buildings).

- File - class for reading/writing files.

- FindList - GameObject search list class.

- FindListItem - FindList search list item class.

- FindItemProperty - the item property class used in FindListItem.

- IgnoreList - ignore list class when searching for GameObjects.

- IgnoreListItem - ignore iist element class when searching for IgnoreList.

- ShopList - shopping list class.

- ShopListItem - ShopList item list class.

- StepCounterObject - the class of the step counter to a tile.

- TextDialog - class of the text dialog object received from the server.

- Shared - the class of the object through which shared memory is managed.

- Sallos - class that implements some of the functionality of the Sallos client.

- CustomGump - a class to work with Custom Gumps.

- CustomGumpResponse - a class for Custom Gump responses. Accessible only from the custom gump callback function.

- PairIntInt - class is an integer-integer value pair.

- PairIntString - class is an integer-string value pair.

- LandTiledataObject - a class with meta information about a land tile.

- StaticTiledataObject - a class with meta information about a static tile.

- VendorList - vendor list class.

- VendorListItem - VendorList item list class.

Global classes (always available - you do not need to create anything additional to use them):

Orion, Player, TextWindow, SelectedTile, Abyss, Shared, Sallos

The global class CustomGumpResponse is only available when the callback is automatically called for this gump.

The include system:

At the top of a script file, write //#include path/to/file

If the syntax is correct, the line will be highlighted in red.

Includes are taken into account when loading/saving/reloading the current script file and when starting a new script if changes have been made to the code (the code itself is not saved, it is recommended to manually save the files).

There are two possible search systems for the include module:

1 - Search relative to the current file being processed (includes may also contain links to other includes. The search path in the first step is calculated relative to the current module being processed).

2 - Search by the direct path specified in the line (if a relative path is specified, instead of the direct path, the file can be found if it lies in the folder where the application was launched from).

The same module (along the same path) is connected only 1 time.

If the file is not found, a corresponding message will be displayed in the game’s system chat (when saving the script).

An example of including a module in a relative path (from a folder with the current script):

//#include common.oajs

An example of including a module in a relative path (from folder includes):

//#include includes/common.oajs

An example of including a module in a relative path (the directory above where the folder is located includes):

//#include ../includes/common.oajs

An example of including a module in a direct way:

//#include E:/OrionScripts/includes/common.oajs

Implemented support for secure includes.

In the context menu of the script editor, you can select the Make protected include file item, and select the file you want to protect.

A file will be created with the extension *.jsp or *.oajsp

This file can be used as a include:

//#include common.oajsp

1.2 Ingame commands

In-game Commands

List of in-game commands. Within game chat, type the command prefix (like underscore or bracket) plus the command (with arguments, if necessary).

For example: _showjournal 15

- info - Get information about an object. Possible arguments: serial

- infotile - Get information about a tile. Possible arguments: lasttile

- saveconfig - Save the assistant profile

- setdressbag - Change target bag for equipment. Possible arguments: serial

- unsetdressbag - Discard a bag for clothes

- getstatus - Get the status of an object from the server. Possible arguments: serial

- click - Click on an object. Possible arguments: serial

- useobject - Use an object. Possible arguments: serial

- attack - Attack an object. Possible arguments: serial

- usetype - Use an object by type within a container. Possible arguments: graphic color container recurse

- usefromground - Use an object by type, on the ground. Possible arguments: graphic color distance

- textopen - Open a text window

- textclose - Close the text window

- textclear - Clear the text window

- textprint - Display text in a text box. Possible arguments: text

- terminate - Stop the specified script. Argument: scriptName

- setlight - Turn on / off lighting. Possible arguments: state value

- setweather - Change the weather. Possible arguments: state index effectsCount temperature

- setseason - Change the season. Possible arguments: state index musicIndex

- cast - Cast a spell. Possible arguments: spellIndexOrName

- useskill - Activate a skill. Possible arguments: skillIndexOrName

- helpgump - Request the 'help' gump from the server

- closeuo - Close the game client

- warmode - Change the battle mode. Possible arguments: state

- morph - Change the character’s body. Possible arguments: graphic

- resend - Request for synchronization with the server

- sound - Play a sound. Possible arguments: index

- emoteaction - Trigger an emote. Possible arguments: actionName

- hide - Hide an object. Possible arguments: serial

- blockmoving - Prohibit character movement. Possible arguments: state

- drop - Discard an item. Possible arguments: serial count X Y Z

- drophere - Drop an object at your feet. Possible arguments: serial count

- moveitem - Move an item. Possible arguments: serial count container X Y Z

- setarm - Remember a set of weapons. Possible arguments: setName

- unsetarm - Forget the weapon set. Possible arguments: setName

- setdress - Remember a set of clothes. Possible arguments: setName

- unsetdress - Forget the set of clothes. Possible arguments: setName

- savehotkeys - Save hotkeys to a file. Possible arguments: fileName

- loadhotkeys - Load hotkeys from a file. Possible arguments: fileName

- track - Show / hide the quest arrow. Possible arguments: state X Y

- infomenu - Display information about a menu

- showjournal - Show information about the journal. Possible arguments: linesCount

- clearjournal - Clear the journal. Possible arguments: pattern serial color type

- journalignorecase - Change the ignore case flag for searching the journal. Possible arguments: state

- ignorereset - Reset the ignore flag from all objects in the world

- infogump - Display information about all active gumps
